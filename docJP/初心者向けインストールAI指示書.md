# Rashinban for AI セットアップ支援プロンプト

> このファイルは、Rashinban for AI のセットアップに不慣れな人が、AIアシスタント（Claude Code、ChatGPT、その他のAIチャットなど）に渡して使うためのものです。
> 使い方：このファイル全体をコピーして、AIとの会話に貼り付けてください。

---

## AIへの指示

あなたはこれから、ユーザーが「Rashinban for AI」というツールをセットアップするのを手伝います。ユーザーはプログラミング初心者を想定しています。以下の方針で進めてください。

- **実行環境がある場合**（ターミナルやファイルを直接操作できる場合）：コマンドを実行する前に、何をするコマンドか一言で説明してから実行してください。
- **実行環境がない場合**（チャットでの会話のみの場合）：コマンドをそのままコピーして使えるコードブロックで提示し、実行後の結果（ターミナルに表示された内容）を貼ってもらってください。**ただし認証トークンが含まれるログを貼ってもらう場合は、トークンの行だけ伏せるよう案内してください**（例：`認証トークン: （伏せました）` のように置き換える）。
- 専門用語（WebSocket、トークン、ポート等）が出てきたら、一言だけ平易な説明を添えてください。長い解説は不要です。
- エラーが出ても、ユーザーを焦らせないでください。「よくあることなので大丈夫です」と一言添えてから対処に進んでください。
- 各ステップが終わるごとに、次に何をするか・なぜそれをするかを簡潔に伝えてください。

以下がセットアップ手順です。ユーザーの状況（OS、Node.jsの有無など）を最初に確認してから、順番に進めてください。

## このセットアップの全体像

次の順番で進めます。

1. 使用するAIエージェント環境（LM Studio等）でAIとチャットできる状態にする
2. Rashinban for AIのプロジェクトフォルダを用意する
3. Chrome拡張機能を読み込む
4. MCPサーバーを一度起動してトークンを確認する
5. 拡張機能にトークンを設定する
6. 使用するAIエージェント環境にMCPサーバーを登録する
7. Webページ上の要素を取得できるか確認する

## よく出てくる用語（先に知っておくと安心）

| 用語 | 意味 |
|---|---|
| AIエージェント環境 | LM StudioやClaude Desktopなど、AIにツールを使わせるためのアプリや実行環境 |

| ポート | パソコン内で通信を受け付ける番号（このツールでは3765と3766の2つ） |
| WebSocket | ブラウザの拡張機能とサーバーがつながるための通信方式 |
| トークン | 接続を許可するための合言葉のようなもの |

## 前提条件の確認

最初に以下を確認してください。

1. **Rashinban for AIのプロジェクトフォルダがあるか**
   READMEなどの案内に従って、Rashinban for AIのプロジェクトフォルダを用意してください。
   ZIPで入手した場合は、先に解凍してください。
   例：`C:\Users\<ユーザー名>\Downloads\Rashinban-for-AI`
   このフォルダの場所を確認してから次に進んでください（後の手順で何度も使います）。

2. **Node.jsがインストールされているか**
   確認コマンド：`node -v`
   インストールされていなければ、[nodejs.org](https://nodejs.org/) から案内してください（LTS版を推奨）。

3. **Google Chromeを使っているか**
   このツールはChrome拡張機能のため、Chromeが必要です。

4. **何のAIエージェント環境と組み合わせる予定か**
   LM Studioなのか、Claude Desktopなのか、他のMCP対応クライアントなのか確認してください。後の手順（MCPサーバーへの登録方法）が変わります。

   > ⚠️ **LM Studio自体がまだインストールされていない、またはAIモデルとまだチャットできない場合**：先に `LMStudioセットアップガイド.md` を参照し、そちらを完了させてから戻ってきてください。このファイルはRashinban for AI固有の手順のみを扱っており、LM Studio自体の使い方は含まれていません。

## セットアップ手順

### ステップ1：Chrome拡張機能を読み込む

1. Chromeで `chrome://extensions` を開く
2. 右上の「デベロッパーモード」をONにする
3. 「パッケージ化されていない拡張機能を読み込む」をクリック
4. プロジェクトフォルダ内の `chrome-extension` フォルダを選択する
   （例：`C:\Users\<ユーザー名>\Downloads\Rashinban-for-AI\chrome-extension`）

> ⚠️ よくある質問：「読み込んだのに何も起きない」→ 正常です。これだけではまだ何も動きません。次のステップに進んでください。

### ステップ2：ターミナルでプロジェクトフォルダを開き、依存パッケージをインストールする

まず、ターミナル（PowerShellやコマンドプロンプト）でプロジェクトフォルダに移動します。

```
cd "C:\Users\<ユーザー名>\Downloads\Rashinban-for-AI"
cd "mcp-server"
npm install
```

これは、サーバーを動かすために必要な部品をダウンロードする作業です。少し時間がかかることがあります。

### ステップ3：MCPサーバーを起動する

```
node server.js
```

成功すると、以下のようなログが表示されます。

```
[Rashinban] === セキュリティ設定 ===
[Rashinban] 認証トークン: （ここに文字列が表示されます）
[Rashinban] 承認済み拡張ID: （初回接続で自動登録）
[Rashinban] =======================
[Rashinban] WebSocketサーバー起動: ws://127.0.0.1:3765
[Rashinban] MCPサーバー起動: http://127.0.0.1:3766/mcp
```

> ⚠️ **重要：表示された「認証トークン」をコピーして控えてください。** これは合言葉のようなもので、次のステップで使います。**他人に見せたり、AIとの会話にそのまま貼り付けたりしないでください。** もしこのログをAIに貼って確認してもらいたい場合は、認証トークンの行だけ `認証トークン: （伏せました）` のように書き換えてから貼ってください。

> ⚠️ **このターミナルの画面は閉じないでください。** サーバーが動いている間は、この画面（黒い画面）を開いたままにしておく必要があります。このツールを使っている間はずっと、サーバーはこのまま動かしておきます（停止したいときは `Ctrl + C`、次に使うときはまた `node server.js` で起動します）。AIエージェント環境がサーバーを起動してくれる方式ではないので、「サーバーを自分で起動しておく → AIエージェントがそこに接続する」という流れになります。

### ステップ4：拡張機能にトークンを設定する

1. `chrome://extensions` で「Rashinban for AI」の「詳細」を開く
2. 「拡張機能のオプション」を開く（見つからない場合は、Chrome右上のパズルピースのアイコン→対象拡張の点3つ（⋮）→「オプション」からも開けます）
3. ステップ3で控えたトークンを貼り付けて保存する

保存できたら、**サーバーは動かしたまま**、ステップ5に進みます。

### ステップ5：AIエージェント環境に登録する

どの環境でも、登録するのは同じ接続先です：`http://127.0.0.1:3766/mcp`
（これは「自分のパソコンの中だけで通じる住所」のようなもので、インターネットには公開されません）

ユーザーが使っているAIエージェント環境によって手順が変わります。

**LM Studioの場合：**

> ⚠️ 以下の設定ファイルの場所や記述方法は、LM Studioのバージョンによって変わる可能性があります。うまくいかない場合は、LM Studio公式のドキュメントもあわせて確認してください。

`C:\Users\<ユーザー名>\.lmstudio\mcp.json` を開く（なければ新規作成）。

```json
{
  "mcpServers": {
    "rashinban-for-ai": {
      "url": "http://127.0.0.1:3766/mcp"
    }
  }
}
```

保存後、LM Studioを再起動してください。

**Claude Codeの場合：**

ターミナルで以下を1行実行するだけです。

```
claude mcp add --transport http rashinban http://127.0.0.1:3766/mcp
```

**Claude Desktopの場合：**

「設定 → コネクタ → カスタムコネクタを追加」で `http://127.0.0.1:3766/mcp` を登録してください。

**その他のMCPクライアントの場合：**

そのクライアントの「リモートMCPサーバー（HTTP）」の登録方法に従い、URLに `http://127.0.0.1:3766/mcp` を指定してください。

> ⚠️ 古い解説記事などで「`command`に`node`、`args`に`server.js`のパスを書く」という設定方法を見かけるかもしれませんが、それは旧バージョン（v0.4.x以前）の方式です。現在のバージョンでは使えません。

## 動作確認

> ⚠️ **使用するモデルのコンテキストウィンドウは8192以上を推奨します。** 小さい値（2048〜4096等）のままだと、取得した要素データが途中で切れ、AIが具体的な要素名や座標を答えられないことがあります。

1. 何か適当なWebページを開き、リロード（F5）する
2. 拡張機能アイコンをクリックして、ポップアップを開く

> ⚠️ **「ポップアップのインジケーターが緑にならない」場合**：サーバーが起動しているか（ステップ3のターミナルが開いたままか）と、トークンが正しく保存されているか（ステップ4）を確認してください。サーバーが動いていてトークンが合っていれば、緑のままになるのが正常です。

3. AIエージェント側（LM Studio等）で、AIに次のように話しかけてみる：
   「今開いているページの操作可能な要素を `get_ui_elements` で確認して」
4. ページの操作可能な要素がリストで返ってくれば成功です

## エラーが出たら

このプロジェクトの `トラブルシューティング_JP.md`（または `Troubleshooting_EN.md`）も一緒にAIに渡してください。よくあるエラーと対処法がまとまっています。

特に多いのは以下のパターンです。

- **「ポートが使用中です」エラー**：前回起動したサーバーが残ったまま。一度停止してから再起動が必要です。
- **`トークン不一致`エラー**：拡張機能をリロードした後、開いている全タブもリロードし忘れていることが多いです。
- **AI側に道具（ツール）が出てこない**：サーバーが起動していない（ステップ3）か、登録したURLが間違っている（`/mcp` まで含めて `http://127.0.0.1:3766/mcp`）ことが多いです。
