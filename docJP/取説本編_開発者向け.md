# Rashinban for AI 取扱説明書

## これは何か

Rashinban for AIは、Webページ内の操作可能なUI要素（CSSセレクタ・座標・状態）を抽出し、ローカルのMCPサーバー経由でAIエージェントに渡すChrome拡張機能です。AIがブラウザを操作する際、スクリーンショットやDOM全文を解析する必要がなくなり、トークン消費と応答時間を削減できます。

構成は2つのパーツに分かれています。

- **Chrome拡張機能**（`chrome-extension/`）：ページを監視し、UI要素を抽出する
- **ローカルMCPサーバー**（`mcp-server/`）：拡張機能とAIエージェントの橋渡しをする

## 必要なもの

- Node.js（インストール済みであること）
- Google Chrome
- ローカルで動作する、MCP対応のAIエージェント環境（LM Studio等）

## インストール手順

### 1. Chrome拡張機能を読み込む

1. Chromeで `chrome://extensions` を開く
2. 右上の「デベロッパーモード」をONにする
3. 「パッケージ化されていない拡張機能を読み込む」をクリック
4. `chrome-extension` フォルダを選択

### 2. MCPサーバーの依存パッケージをインストールする

プロジェクトフォルダ直下で以下を実行します。

```powershell
cd "mcp-server"
npm install
```

### 3. MCPサーバーを起動する

```powershell
node server.js
```

以下のようなログが表示されれば起動成功です。

```
[Rashinban] === セキュリティ設定 ===
[Rashinban] 認証トークン: <トークンが表示される>
[Rashinban] 承認済み拡張ID: （初回接続で自動登録）
[Rashinban] =======================
[Rashinban] WebSocketサーバー起動: ws://127.0.0.1:3765
[Rashinban] MCPサーバー起動: http://127.0.0.1:3766/mcp
```

**表示された認証トークンを控えてください**（次の手順で使います）。

> ポートは2つ使います。競合する場合は該当箇所を書き換えてください。
> - **3765**（拡張機能⇔サーバー間 WebSocket）：`mcp-server/server.js` 内の `WS_PORT` と `chrome-extension/content.js` 内の `WS_URL` の両方を同じ番号に書き換え、サーバー再起動と拡張機能リロードを行う。
> - **3766**（AIエージェント⇔サーバー間 HTTP）：`mcp-server/server.js` 内の `HTTP_PORT` を書き換え、各AIエージェントの接続URLも合わせて変更する。

### 4. 拡張機能にトークンを設定する

1. `chrome://extensions` で Rashinban for AI の「詳細」を開く
2. 「拡張機能のオプション」を開く
3. 控えたトークンを貼り付けて保存する

トークンを拡張機能に保存できたら、**サーバーはそのまま起動したままにしてください**。v0.5.0以降、サーバーは常駐型です。AIエージェントがサーバーを起動するのではなく、起動済みのサーバーへ各エージェントがHTTPで接続します。この方式により、**複数のAIエージェントが同時に接続・利用できます**。

### 5. AIエージェントを接続する

接続先URLはどのクライアントでも共通です：`http://127.0.0.1:3766/mcp`

**LM Studio の場合**：`C:\Users\<ユーザー名>\.lmstudio\mcp.json` を開き（なければ新規作成）、以下を記述して LM Studio を再起動します。

```json
{
  "mcpServers": {
    "rashinban-for-ai": {
      "url": "http://127.0.0.1:3766/mcp"
    }
  }
}
```

**Claude Code の場合**：ターミナルで以下を実行します。

```powershell
claude mcp add --transport http rashinban http://127.0.0.1:3766/mcp
```

**Claude Desktop の場合**：「設定 → コネクタ → カスタムコネクタを追加」で上記URLを登録します。カスタムコネクタが使えない環境では、`claude_desktop_config.json` に `mcp-remote` を使ったブリッジ設定でも接続できます。

```json
{
  "mcpServers": {
    "rashinban-for-ai": {
      "command": "npx",
      "args": ["mcp-remote", "http://127.0.0.1:3766/mcp"]
    }
  }
}
```

> **旧バージョン（v0.4.x以前）からの移行**：以前の「`command`に`node`、`args`に`server.js`のパス」というstdio方式の設定は使えなくなりました。各クライアントの設定を上記のURL方式に置き換えてください。

## 基本的な使い方

1. MCPサーバーを起動した状態で、操作したいWebページを開く
2. ページがリロードされていることを確認する（拡張機能を読み込んだ直後に開いていたタブには反映されません）
3. AIエージェント（LM StudioのDeveloperモード等）から `get_ui_elements` ツールを呼び出す
4. ページ内の操作可能要素（セレクタ・座標・状態）がJSON形式で返ってくる

## マルチエージェント同時利用（v0.5.0〜）

サーバーは複数のAIエージェントからの同時接続に対応しています。

- **セッション分離**：エージェントごとにMCPセッションが分かれ、互いの状態は混ざりません
- **DOM変化の複製配信**：ページの変化は接続中の全エージェントに配信されます。あるエージェントが `get_dom_changes` で取得しても、他のエージェントのキューは消費されません
- **道標のエージェント別保存**：`save_michishirube` に `agent_id` を指定すると、エージェントごとに独立して道標を保存・取得できます。省略時はセッション固有のIDが使われます（セッションが切れると変わるため、**引き継ぎ用途では明示指定を推奨**）
- **タブの指定**：`get_ui_elements` に `clientId`（`list_connected_tabs` で取得）を渡すと、エージェントAはタブ1、エージェントBはタブ2、と別々のタブを並行操作できます

## 利用できるツール（MCP）

| ツール名 | 内容 |
|---|---|
| `get_ui_elements` | 操作可能UI要素を取得する。`clientId` で対象タブを指定可能（省略時は最後に応答したタブ） |
| `list_connected_tabs` | 現在接続中のブラウザタブ一覧（clientId・URL・タイトル）を取得する |
| `get_dom_changes` | ページの動的な変化（新しく出現した要素）を取得する。自分のセッションのキューだけが消費される |
| `save_michishirube` | 道標をエージェント別に保存する（`agent_id` 省略時はセッション固有ID） |
| `get_latest_michishirube` | 道標を取得する。`agent_id` 指定で他エージェントの道標も参照可能 |
| `list_michishirube` | 保存されている道標の一覧（agent_id・保存日時・goal・status）を返す |
| `clear_michishirube` | 道標を削除する。`all: true` で全エージェント分を削除 |

## 動作確認方法

### コンソールログを見る

対象ページで DevTools（F12）→ Console タブを開くと、抽出結果がログとして出力されます。

```
[Rashinban] 42個の操作可能要素を検出 - https://...
  enabled: 38 / disabled: 2 / hidden: 1 / その他: 1
```

### ポップアップの接続状態を見る

拡張機能アイコンをクリックすると、接続状態インジケーターが表示されます。緑＝接続中です。

> v0.5.0以降はサーバーが常駐型になったため、サーバーが起動していてトークン設定が正しければ、インジケーターは緑（接続中）のままになります。「未接続」のままの場合は、サーバーが起動しているか・トークンが一致しているかを確認してください。

### サーバー側のログを見る

MCPサーバーを起動しているターミナルに、接続・DOM変化などのログが流れます。

```
[Rashinban] 認証成功: <拡張ID> → <URL>
[Rashinban] DOM変化を受信: 6個の新要素 (https://...)
```

## 言語選択

拡張機能のポップアップとオプション画面は、5言語（日本語・英語・中国語（簡体字）・スペイン語・フランス語）に対応しています。どちらの画面でもドロップダウンから言語を選択でき、選択内容は両画面で共有されます。

## フィードバックの送り方

ポップアップとオプション画面の両方に、フィードバック用ボタンがあります。クリックすると新しいタブでフィードバックフォームが開きます。バグ報告・機能要望・その他の感想、何でも歓迎です。

## セキュリティについて

本ツールは3層のセキュリティで保護されています。

1. **localhost限定バインド**：外部ネットワークからの接続を受け付けません
2. **拡張機能ID検証**：初回接続時に自動承認・記録され、以降は照合されます
3. **トークン認証**：拡張機能とMCPサーバー間の通信を認証します

これらの設定は `mcp-server/config.json` に保存されます。このファイルには認証トークンが含まれるため、**他人と共有したり、リポジトリにコミットしたりしないでください**（`.gitignore` で除外済みですが、zip圧縮やフォルダコピーでの共有では除外されない点に注意してください）。
